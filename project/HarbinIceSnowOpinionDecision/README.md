

# ❄️ 哈尔滨冰雪大世界：舆情决策系统

本项目是一款基于 **LangGraph** 状态机架构和 **FAISS** 向量数据库开发的“数字孪生”舆情演化沙盘。系统专为哈尔滨冰雪大世界旅游危机管理设计，通过多智能体（Multi-Agent）协同辩论与 RAG 检索增强技术，将海量原始舆情数据转化为可落地的应急响应方案。

## 🚀 项目核心内容 (Project Content)

### 1. 语义感知引擎 (Vector RAG)
系统集成 **DashScope Text-Embedding** 技术，实现了超越关键词匹配的深度语义理解。当突发危机发生时，系统能自动识别“排队受罪”与“现场管理缺失”之间的关联，并从历史库中检索最具对标价值的案例。

### 2. 多智能体分层决策 (Multi-Agent Hierarchical Logic)
系统模拟真实景区的应急决策链，由四个垂直职能的智能体协同作业：

* 🛠️ **运营总监 (Op Agent)**：输出现场即时管控建议，侧重人力分流与物资调配。
* 💰 **财务总监 (Fin Agent)**：根据情绪极性核算退赔预算，进行资金风险管控。
* 📢 **公关总监 (PR Agent)**：融合各部门意见，撰写符合品牌基调的正式对外通稿。
* 🌐 **压力仿真 (Supervisor)**：模拟网民矩阵进行量化评分，预测方案发布后的社会反馈。

### 3. 动态发酵预测曲线 (Visual Simulation)
结合 Plotly 动态渲染技术，系统会根据推演分数生成未来 72 小时的“负面压力演化曲线”，直观展示危机从爆发到回归“安全区”的预测轨迹。

## 🛠️ 环境准备与部署 (Getting Started)

### 1. 环境准备
* **Python 版本**：推荐使用 Python 3.10 或以上版本。
* **API 密钥**：本系统依赖通义千问大模型，请准备好 `DashScope API Key`。
* **💡 温馨提示**：通义千问在处理中文旅游语境（如“尔滨梗”）时理解更为细腻，且响应极其迅速。
### 2. 本地部署步骤
在终端（Terminal）中依次执行以下指令：

1.  **克隆/下载**本项目至本地文件夹。
2.  **安装核心依赖**：
    ```bash
    pip install streamlit pandas plotly scikit-learn dashscope faiss-cpu langchain-community langchain-openai langgraph python-dotenv
    ```
3.  **配置数据**：
     `data/` 文件夹内的舆情数据。
4.  **启动沙盘**：
    ```bash
    streamlit run main.py
    ```

## 💻 本地运行说明 (Execution Guide)

### 1. 首次初始化
* 系统启动后，请在侧边栏输入您的 API Key。
* **注意**：首次运行会显示 🚀 `正在分批构建索引...`。系统会将 1900 条数据分为若干批次进行向量化处理，耗时约 1-3 分钟。
* 构建完成后，本地会生成 `faiss_local_index` 文件夹。后续所有启动都将实现 0.1 秒极速加载。

### 2. 闭环干预 (Human-in-the-loop)
* 若对生成的公关通稿不满意，可直接在底部聊天框发送修改指令（例如：“态度再诚恳一点”）。
* 系统将触发 rework 逻辑，各部门智能体会基于新指令重新流转并更新方案。

### 3. 缓存刷新
* 若您在 `data` 中新增了原始数据，请手动删除 `faiss_local_index` 文件夹。系统重启后会重新扫描并构建最新的语义“大脑”。
